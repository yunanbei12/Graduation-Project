# 4.4.3 AI咨询功能演示文档

## 1. 功能定位

AI咨询功能是本系统智能服务模块的重要组成部分，主要用于处理课程推荐、订单查询、课包查询、优惠券查询、退款说明以及人工转接等常见咨询场景。与传统静态问答不同，本系统并未采用完全依赖大模型自由生成回复的方式，而是结合会话管理、关键词意图识别、知识库匹配、业务数据查询以及人工接管机制，形成了“规则判断为主、模型润色为辅”的实现方案。该方案既保证了订单、课包、优惠券等业务查询结果的准确性，又提升了回复内容的自然性和交互体验。

---

## 2. 演示目标

本节演示的目标主要包括以下四点：

1. 展示用户如何在小程序端通过AI客服发起课程咨询。
2. 展示系统如何识别用户问题意图，并结合真实业务数据生成回复。
3. 展示AI咨询结果如何返回结构化卡片和操作按钮。
4. 展示系统如何支持人工接管与后台会话管理。

---

## 3. 演示准备

为保证演示过程顺利，建议提前准备以下测试数据：

1. 一个已登录测试用户，账号下最好有订单、课包或优惠券数据。
2. 若要展示游客咨询场景，可额外准备一个未登录状态的客户端。
3. 知识库中提前录入若干条常见问题内容，例如退款说明、课程咨询和优惠券规则。
4. 后台准备一个管理员账号，用于展示会话管理和人工接管。

---

## 4. 演示步骤

### （1）课程推荐演示

演示时可按如下顺序进行：

1. 在小程序首页点击“AI客服”或“咨询顾问”入口。
2. 输入问题，例如“推荐一个适合新手减脂的课程”。
3. 展示系统返回的推荐文字和课程卡片。
4. 点击课程卡片，展示可以直接跳转到课程详情页。

这一过程可重点说明：系统并不是返回固定文本，而是先识别“课程推荐”意图，再从真实课程数据中提取结果，并通过结构化卡片方式返回前端。

### （2）业务查询演示

演示时可继续展示AI与真实业务数据的联动能力：

1. 在聊天页输入“我的课包还剩几节”或“我有哪些可用优惠券”。
2. 展示系统返回与当前账号真实业务状态一致的查询结果。
3. 如前面已完成订单支付或销课演示，可再次提问，展示AI回复会随业务数据变化而更新。

### （3）人工接管演示

演示时可按如下顺序进行：

1. 在小程序端输入“我要人工客服”。
2. 展示当前会话状态切换为人工处理中。
3. 切换到后台“AI会话管理”页面。
4. 展示管理员可以看到该会话、查看历史消息并发送人工回复。
5. 返回小程序端刷新聊天页，展示人工回复结果已同步显示。

---

## 5. 核心实现说明

### （1）会话创建与消息接收

用户在小程序端发送咨询消息后，系统首先对输入内容进行非空校验，并根据会话编号、登录用户标识或游客标识准备当前会话。若当前不存在有效会话，则自动创建新会话；若用户此前以游客身份咨询，后续登录后系统还可将游客会话归并到当前用户名下。完成会话准备后，系统立即保存用户消息，为后续意图识别和回复生成提供输入基础。其处理逻辑如下：

```java
chat(request, userId) {
    // 1. 校验用户输入消息是否为空
    validateMessage(request.message);

    // 2. 根据 sessionId、userId 或 guestToken 准备当前会话
    AiSession session = prepareSession(request.sessionId, userId, request.guestToken);

    // 3. 保存用户发送的消息
    saveUserMessage(session.id, userId, request.message);
}
```

这一阶段的关键在于通过统一的会话准备逻辑处理新建会话、续用旧会话以及游客会话归并问题，从而保证多轮咨询的连续性。

### （2）意图识别与业务分发

在用户消息保存完成后，系统需要判断当前问题属于哪一类业务咨询，并进入对应处理分支。由于本系统咨询场景相对固定，主要集中在订单、课包、优惠券、课程推荐和售后说明等类别，因此采用关键词匹配方式实现轻量级意图识别。识别完成后，系统根据意图构造回复草稿，并分发到不同的业务处理逻辑中。其处理逻辑如下：

```java
reply(message, userId, sessionId) {
    // 1. 识别用户问题意图
    String intent = detectIntent(message);

    // 2. 估算当前识别结果的置信度
    BigDecimal confidence = estimateConfidence(intent, message);

    // 3. 根据意图进入不同业务处理分支
    ChatDraft draft = buildReply(intent, message, userId, sessionId);

    return draft;
}
```

其中，系统当前支持课程推荐、排课查询、订单查询、课包查询、优惠券查询、退款说明、账号帮助、人工客服和通用问答等多个意图类型。

### （3）业务查询与知识增强回复

在具体回复生成阶段，系统会结合真实业务数据构造回复内容。例如，订单查询会读取用户最近订单记录，课包查询会读取剩余课时和有效期，优惠券查询会获取当前可用优惠券信息。对于课程推荐和通用问答场景，系统还会进一步匹配知识库内容，并调用模型服务对回复文本进行润色；若模型调用失败，则自动回退到规则模板回复。其核心逻辑如下：

```java
applyFinalReply(draft, intent, message, userId, knowledges, cards, fallback) {
    // 1. 根据当前意图查询业务数据或知识库内容
    loadBusinessData(intent, userId);
    matchKnowledges(intent, message);

    // 2. 调用模型服务生成更自然的回复话术
    String modelReply = generateReply(intent, message, userProfile, knowledges, cards);

    // 3. 模型可用时优先使用润色结果，否则回退到规则模板
    draft.replyText = hasText(modelReply) ? modelReply : fallback;
}
```

该实现表明，本系统AI咨询并不是直接依赖大模型自由生成结果，而是先由规则和业务数据给出可靠答案，再由模型优化表达方式，从而兼顾了业务准确性与交互自然性。

### （4）回复保存与人工转接控制

在回复内容生成后，系统会将AI回复写入消息记录，并同步更新会话摘要信息，包括最后问题、最后回复、最后意图和最后消息时间等字段，以保证会话状态的完整性。当用户主动请求人工客服时，系统会创建转人工记录，并将当前会话切换为人工处理状态；此后用户继续发送的消息将不再触发AI自动回复，而是仅写入消息记录，并由后台客服工作台通过轮询方式同步显示。其核心逻辑如下：

```java
finishReply(session, message, intent, confidence, draft) {
    // 1. 构造并保存AI回复消息
    saveAssistantMessage(session.id, intent, confidence, draft);

    // 2. 更新会话摘要信息
    updateSessionSnapshot(session, message, draft);

    // 3. 若用户请求人工客服，则创建转人工记录并切换会话状态
    if (draft.needHandover) {
        createHandover(session.id);
        markSessionWaitingManual(session.id);
    }
}
```

### （5）后台会话与知识库管理

除了前台咨询处理外，系统后台还提供了AI会话管理、知识库管理和统计分析能力。管理员可以在后台查看咨询会话、按状态筛选问题、查看人工转接记录以及维护知识条目内容，从而实现对AI咨询模块的统一管理。其核心逻辑如下：

```java
sessionList(pageNum, pageSize, keyword, intent, status, needHandover) {
    // 1. 按条件分页查询后台会话列表
    Page page = getAdminSessionPage(pageNum, pageSize, keyword, intent, status, needHandover);

    // 2. 返回会话摘要信息，供后台管理页统一展示
    return page;
}
```

## 6. 演示讲解重点

正式展示时，建议重点强调以下内容：

1. AI咨询并不是通用闲聊，而是面向课程、订单、课包和优惠券等真实业务场景设计的。
2. 系统采用“规则识别 + 业务查询 + 知识库 + 模型润色”的混合方案，既保证准确性，又保证表达自然。
3. AI回复支持结构化卡片和跳转动作，说明它不仅能回答，还能引导用户继续完成业务操作。
4. 人工接管机制保证了系统在复杂问题场景下仍具有可持续服务能力。

---

## 7. 总结

AI咨询功能并不是简单的聊天问答，而是一个融合会话管理、规则路由、知识匹配、业务查询、模型润色和人工协同处理的综合服务模块。该模块既保证了咨询结果与真实业务数据的一致性，又通过知识库与模型增强机制改善了用户体验，同时借助后台会话管理和人工转接能力形成了较完整的智能服务闭环。
