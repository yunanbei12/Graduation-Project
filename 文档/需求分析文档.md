## 一、系统概述

### 1.1 系统定位
一个面向体育教培机构的 SaaS 管理系统，支持**私教上门**和**团课**两种业务模式，融合**拼团营销**、**智能推荐**、**AI 客服**三大特色功能。

### 1.2 核心特色
1. **上门体育模式**：支持教练上门教学，地点由学员决定
2. **双模式拼团**：私教课包拼团（价格优惠）+ 团课拼团（成团上课）
3. **智能推荐**：基于规则混合推荐算法
4. **AI 客服**：接入大模型 API 的智能问答系统

### 1.3 技术栈
- 前端：Vue3 + 微信小程序
- 后端：SpringBoot3
- 数据库：MySQL
- AI 服务：大模型 API（文心一言/通义千问）

### 3.1 营销管理模块

#### 3.1.1 拼团功能（核心特色）

**业务场景 1：私教课包拼团**

```
场景：张三发起"10 节私教课"拼团，原价 2000 元，拼团价 1500 元

流程：
1. 张三支付 1500 元发起拼团（选择课程包：10 节课）
2. 设置拼团参数：需 3 人成团，有效期 48 小时
3. 分享小程序码到微信群/朋友圈
4. 李四、王五看到后参与拼团并支付
5. 48 小时内满 3 人 → 成团 → 三人各自获得 10 节课时
6. 48 小时内未满 3 人 → 拼团失败 → 自动原路退款
```

**业务场景 2：团课拼团**

```
场景：后台创建"周六下午幸福小区羽毛球团课"

流程：
1. 后台创建团课：时间（周六 15:00）、地点（幸福小区羽毛球场）、教练、需 5 人成团、单价 50 元
2. 小程序展示该团课，用户可报名参团
3. 用户支付报名（可先收款，不成团再退）
4. 成团判断：
   - 成功：微信订阅消息通知上课时间地点
   - 失败：自动取消课程 + 自动原路退款
5. 后台也可手动取消团课（触发自动退款）
```

**拼团核心参数**：

| 参数 | 私教课包拼团 | 团课拼团 |
|------|--------------|----------|
| 成团人数 | 2-5 人可配置 | 3-10 人可配置 |
| 有效期 | 24/48/72 小时可选 | 开课前 2 小时截止 |
| 优惠方式 | 折扣价/满减 | 团购价 |
| 退款规则 | 失败自动原路退款 | 失败自动原路退款 |
| 课时发放 | 成团后各自到账 | 成团后统一上课 |

**功能清单**：
- [ ] 用户发起拼团（选择课程包/团课）
- [ ] 用户参与拼团（查看拼团详情、剩余名额、倒计时）
- [ ] 拼团分享（小程序码、分享文案）
- [ ] 拼团状态监控（拼团中/成功/失败）
- [ ] 成团自动处理（课时发放/课程确认）
- [ ] 失败自动退款（调用支付接口原路退回）
- [ ] 后台拼团列表查看
- [ ] 后台手动取消团课（触发退款）

### 3.5 智能推荐模块（毕设亮点）

#### 3.5.1 推荐算法设计

**混合推荐公式**：
```javascript
recommend_score = 
  popularity_score * 0.4 +  // 热度分
  personalization_score * 0.4 +  // 个性化分
  operation_weight * 0.2  // 运营权重
```

**1. 热度分计算**：
```python
popularity_score = 
  (recent_sales / max_sales) * 0.5 +  // 近 7 天销量归一化
  (favorites / max_favorites) * 0.3 +  // 收藏量归一化
  (view_count / max_views) * 0.2  // 浏览量归一化
```

**2. 个性化分计算**：
```python
# 用户画像标签
user_profile = {
  'preferred_type': '私教课',  # 历史购买最多的类型
  'price_range': (1000, 3000),  # 可接受价格区间
  'time_preference': '周末',  # 偏好上课时间
  'location': '幸福小区',  # 居住区域
  'sport_preference': '篮球'  # 偏好运动项目
}

# 课程匹配度
personalization_score = 
  type_match * 0.3 +  # 课程类型匹配
  price_match * 0.3 +  # 价格区间匹配
  location_match * 0.2 +  # 地理位置匹配
  sport_match * 0.2  # 运动项目匹配
```

**3. 运营权重**：
```
后台可手动设置课程推广权重（1-10）
用于：新课程推广、促销活动、清库存
```

### 3.6 AI 客服模块（毕设亮点）

#### 3.6.1 技术架构

```
方案：接入大模型 API + RAG（检索增强生成）

流程：
1. 用户提问（小程序客服入口）
2. 意图识别（分类：课程咨询/拼团规则/订单问题/其他）
3. 知识库检索（从 FAQ 库检索相关问题）
4. Prompt 构建：
   """
   你是体育教培机构客服助手，请根据以下知识回答问题：
   
   【知识库】
   {retrieved_docs}
   
   【用户问题】
   {user_question}
   
   请给出准确、友好的回答。
   """
5. 调用大模型 API（文心一言/通义千问）
6. 返回答案给用户
7. 记录问答日志
```

## 四、业务流程图

### 4.1 私教课包拼团流程
```
用户 A 发起拼团
  ↓
选择课程包（10 节私教课）
  ↓
支付拼团价 1500 元
  ↓
设置拼团参数（3 人成团，48 小时有效）
  ↓
分享小程序码到微信群
  ↓
用户 B、C 参与拼团并支付
  ↓
[判断] 48 小时内是否满 3 人？
  ├─ 是 → 成团 → 三人各自获得 10 课时 → 微信通知
  └─ 否 → 拼团失败 → 自动原路退款 → 微信通知
```

### 4.2 团课拼团流程
```
后台创建团课排期
  ↓
设置：时间、地点、教练、5 人成团、50 元/人
  ↓
小程序展示团课
  ↓
用户报名支付
  ↓
[判断] 开课前 2 小时是否满 5 人？
  ├─ 是 → 成团 → 发送上课通知 → 上课 → 销课
  └─ 否 → 不成团 → 自动取消课程 → 自动退款
```

### 4.3 销课流程（私教）
```
学员微信私聊教练约课
  ↓
教练上课（约定时间地点）
  ↓
课后微信群通知运营
  ↓
运营后台操作销课
  ↓
选择学员 → 选择课包 → 扣除 1 课时 → 填写备注
  ↓
系统发送微信订阅消息给学员
  ↓
学员小程序查看课时余额和销课记录
```

### 4.4 推荐算法执行流程
```
用户打开小程序首页
  ↓
获取用户画像（历史行为、偏好标签）
  ↓
获取候选课程池（所有上架课程）
  ↓
计算每门课程推荐分数：
  - 热度分（销量、收藏、浏览）
  - 个性化分（类型、价格、位置、项目匹配）
  - 运营权重（后台设置）
  ↓
按分数排序取 TOP5
  ↓
记录推荐日志（用户、课程、分数、原因）
  ↓
展示推荐结果
  ↓
[异步] 记录用户点击/购买行为
  ↓
用于算法效果评估
```

### 4.5 AI 客服问答流程
```
用户打开客服对话框
  ↓
输入问题："私教课怎么收费？"
  ↓
意图识别：课程咨询 → 价格问题
  ↓
知识库检索：检索"课程价格"相关 FAQ
  ↓
构建 Prompt（知识库 + 用户问题）
  ↓
调用大模型 API
  ↓
返回答案："私教课价格因课程类型而异..."
  ↓
展示答案给用户
  ↓
记录问答日志（问题、答案、满意度）
```

## 八、待确认细节问题

### 问题 1：私教课上门费用计算
```
场景：学员购买私教课包，教练上门教学
问题：上门费用如何收取？

选项：
A. 包含在课包价格中（不额外收费）
B. 按次收取上门费（100 元/次，下单时选择）
C. 按距离收费（3km 内免费，超出部分收费）

建议：选 B（简单明确，后台可配置上门费标准）
```

### 问题 2：团课取消政策
```
场景：用户已报名团课，开课前临时取消
问题：是否允许用户主动取消？退款规则？

选项：
A. 不允许取消（保证成团率）
B. 开课前 24 小时可免费取消
C. 扣除手续费后退款

建议：选 B（平衡用户体验和成团率）
```

### 问题 3：课时转让规则
```
场景：学员 A 买了课包，因故想转让给学员 B
问题：是否允许转让？

选项：
A. 不允许转让
B. 允许转让 1 次（防止倒卖）
C. 允许无限次转让

建议：选 B（人性化管理，防止滥用）
```

### 问题 4：推荐算法冷启动
```
场景：新用户首次打开小程序，没有行为数据
问题：如何推荐？

选项：
A. 按热度排序（销量最高）
B. 按运营权重排序（推广课程）
C. 随机推荐 + 引导用户选择偏好

建议：A + B 结合（新用户按热度 + 运营权重）
```

### 问题 5：AI 客服无法回答时的处理
```
场景：用户问题超出知识库范围，大模型也无法准确回答
问题：如何处理？

选项：
A. 提示"抱歉，我无法回答，请联系人工客服"
B. 转接人工客服（微信客服消息）
C. 记录问题，后续补充知识库

建议：A + C（即时响应 + 持续优化）
```
