## 二、当前代码工程情况

当前项目采用三工程结构：

1. `backend/`
   Spring Boot 3 + Java 17 + MyBatis-Plus 后端服务
2. `admin-web/`
   Vue3 + Vite + TypeScript + Element Plus 后台管理端
3. `mini/`
   uni-app 小程序端正式工程

#### 3.1.9 支付模块（新增）

已完成：

1. 支付流水表（payments）
2. 获取支付参数接口（模拟微信支付）
3. 模拟支付回调接口（测试用）
4. 支付成功后订单状态自动流转
5. 商品订单支付后自动扣减库存
6. 商品订单支付后更新销量
7. 支付幂等控制

支付流程说明：

1. **模拟支付模式**：适合毕业设计演示，不接入真实微信支付
2. **支付流程**：
   - 用户点击支付 → 创建支付流水 → 返回支付参数
   - 调用模拟回调接口 → 更新支付状态 → 触发订单履约
3. **订单状态流转**：
   - 支付前：`payStatus=0, orderStatus=0`（待支付）
   - 支付后：`payStatus=1, orderStatus=1`（已支付，待履约）
4. **商品订单履约**：
   - 扣减 SKU 库存
   - 增加 SKU 销量
   - 更新订单明细履约状态为"待发货"
