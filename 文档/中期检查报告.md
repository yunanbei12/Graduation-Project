**（1）混合推荐算法公式**
```python
recommend_score = 
  popularity_score * 0.4 +  # 热度分
```

**（2）热度分计算**
```python
popularity_score = 
  (recent_sales / max_sales) * 0.5 +  # 近 7 天销量归一化
  (favorites / max_favorites) * 0.3 +  # 收藏量归一化

```
- 技术实现：Redis ZSET 存储课程热度值，定时任务每小时更新

**（3）个性化分计算**
```python
# 用户画像标签（从 orders、lessons 表提取）
user_profile = {
  'preferred_type': '私教课',  # 历史购买最多的类型
  'price_range': (1000, 3000),  # 可接受价格区间（分位数计算）
  'time_preference': '周末',  # 偏好上课时间
  'location': '幸福小区',  # 居住区域
  'sport_preference': '篮球'  # 偏好运动项目
}

# 课程匹配度（余弦相似度）
personalization_score = 
  type_match * 0.3 +  # 课程类型匹配（0/1）
  price_match * 0.3 +  # 价格区间匹配（0/1）
  location_match * 0.2 +  # 地理位置匹配（0/1）
  sport_match * 0.2  # 运动项目匹配（0/1）
```
- 技术实现：MySQL 存储用户画像标签，Java 服务实时计算匹配度

### 2. AI 客服模块（核心亮点）

**剩余工作**：
- ❌ 知识库管理系统
- ❌ 大模型 API 接入与 RAG 实现
- ❌ 问答日志与效果评估

**技术方案**：

**（1）RAG 架构（检索增强生成）**
```
流程：
1. 用户提问 → 意图识别（分类：课程/拼团/订单/其他）
2. 知识库检索（基于关键词 + 语义相似度）
3. Prompt 构建：
   """
   你是体育教培机构客服助手，请根据以下知识回答问题：
   
   【知识库】
   {retrieved_docs}
   
   【用户问题】
   {user_question}
   
   请给出准确、友好的回答。
   """
4. 调用大模型 API（文心一言/通义千问）
5. 返回答案 + 记录日志
```

**（2）意图识别**
```python
# 规则分类 + 关键词匹配
intent_keywords = {
  '课程咨询': ['价格', '收费', '课时', '有效期', '地点', '时间'],
  '拼团规则': ['拼团', '成团', '退款', '人数', '分享'],
  '订单问题': ['退款', '转让', '发票', '支付'],
  '上课相关': ['预约', '请假', '课时', '签到'],
  '其他': ['投诉', '建议', '人工客服']
}

# 计算意图分数
intent_score = sum(关键词出现次数) / len(关键词列表)
```
- 技术实现：Java 正则表达式匹配 + 关键词权重计算

**（3）知识库检索**
```python
# 方案 1：关键词检索（简单）
- 使用 MySQL FULLTEXT 全文索引
- SELECT * FROM faq WHERE MATCH(question) AGAINST('用户问题')

# 方案 2：语义检索（推荐）
- 使用 Sentence-BERT 将问题转为向量（768 维）
- 使用 FAISS 存储 FAQ 向量，计算余弦相似度
- 返回 Top3 最相似的 FAQ
```
- 技术实现：
  - 向量模型：`paraphrase-multilingual-MiniLM-L12-v2`（支持中文）
  - 向量检索：FAISS（Facebook 开源向量搜索引擎）
  - 向量存储：本地文件 + 内存索引

**（4）大模型 API 接入**

```python
# 文心一言 4.0
import requests
```
