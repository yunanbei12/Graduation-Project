## 四、请求流向详解

### 4.1 管理后台前端的请求流向

```
管理员操作 → 管理后台前端 → Vite代理 → 后台管理接口(8085) → 业务逻辑层 → 数据库
```

管理后台前端通过 Vite 的代理功能，将所有 `/api/*` 请求转发到后台管理接口（8085）。

**配置文件 `vite.config.js`**：
```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8085',  // 代理到后台管理接口
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/api/, '')
  }
}
```

### 4.2 小程序端的请求流向

```
用户操作 → 小程序 → 直接请求 → 用户端接口(8086) → 业务逻辑层 → 数据库
```

小程序直接请求用户端接口（8086），不经过代理。

**配置文件 `utils/config.js`**：
```javascript
const BASE_URL = 'http://localhost:8086'  // 直接请求用户端接口
```

## 五、后端两个应用的区别

| 对比项 | kinetic-sports-admin (8085) | kinetic-sports-api (8086) |
|--------|----------------------------|---------------------------|
| 服务对象 | 管理后台前端 | 小程序端 |
| 主要功能 | 后台管理、数据看板、AI客服工作台 | 用户登录、浏览、下单、AI咨询 |
| RabbitMQ | `rabbit-enabled: false`（不启用） | `rabbit-enabled: true`（启用） |
| 微信配置 | 无 | 有微信小程序 appid/secret |
| 典型接口 | 课程管理、订单管理、用户管理 | 登录注册、课程列表、下单支付 |

> 两个应用共享同一套业务逻辑层（kinetic-sports-service），连接同一个数据库。

## 十、亮点功能简述

### 10.1 个性化推荐算法

融合多源信号构建用户画像，计算综合得分后排序推荐：

```
最终得分 = 0.45 × 行为相似度
         + 0.25 × 内容相似度
         + 0.20 × 热门度
         + 0.10 × 可用性（团课是否有可约场次）
```

数据来源：已支付订单、购物车、上课签到、浏览/点击行为，越近的行为权重越高（时间衰减）。

### 10.2 AI 客服

采用"意图识别 + 业务数据查询 + 大模型润色"的混合模式：

1. 关键词匹配识别用户意图（订单查询、课包查询、退款帮助等）
2. 查询真实业务数据（订单、课包、优惠券等）
3. 构建结构化回复（文本 + 卡片 + 操作按钮）
4. 可选调用 Kimi 大模型对回复进行润色
5. 复杂问题支持转人工，后台提供客服工作台

### 10.3 订单消息队列

使用 RabbitMQ 延迟队列实现：

- **订单超时自动取消**：下单后发布延迟消息，超时未支付则自动关单
- **团课成团检查**：开课前检查是否达到最低成团人数
- **支付成功后处理**：异步发放满额优惠券等

### 10.4 Redis 分布式锁

使用 Redisson 保证并发安全：

| 锁名称 | 用途 |
|--------|------|
| `order:schedule:check:{id}` | 下单时锁定团课名额检查 |
| `order:pay:{id}` | 支付时锁定订单 |
| `order:schedule:{id}` | 支付时锁定团课名额 |
| `order:sku:{id}` | 扣减商品库存时锁定 |

## 十一、典型数据流示例

### 用户下单流程

```
小程序 → 用户端接口(8086) → OrderService
                                ├── 校验课程/商品/地址/手机号
                                ├── 计算总价与优惠券
                                ├── 创建 Order + OrderItem（写数据库）
                                ├── 锁定优惠券
                                └── 发布延迟关单消息（RabbitMQ）
                                         ↓（超时未支付）
                                    自动取消订单，释放优惠券
```

### AI 客服咨询流程

```
小程序 → 用户端接口(8086) → AiCustomerService
                                ├── 识别意图（关键词匹配）
                                ├── 查询业务数据（MySQL）
                                ├── 构建回复草稿（文本 + 卡片）
                                ├── 可选：调用 Kimi 大模型润色
                                └── 保存消息，返回给用户
```
