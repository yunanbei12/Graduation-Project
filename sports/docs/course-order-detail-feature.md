# 课程订单详情页面功能说明

## 功能概述

为课程订单增加详情页面，展示订单完整信息、课包进度、消课记录等，同时更新订单列表的状态标签，支持新增的"已完成"状态。

## 修改内容

### 1. 课程订单列表页面 (`my-orders-course.vue`)

**状态标签更新**：
- 原状态：全部、待支付、已支付、已取消
- 新状态：全部、待支付、**进行中**、**已完成**、已取消

**状态映射更新**：
```javascript
// 状态文本
1: '待支付'
2: '已支付'
3: '进行中'  // 原"已预约"
4: '已完成'  // 新增
5: '已取消'
6: '退款中'
7: '已退款'
8: '退款驳回'

// 状态样式
paid: 蓝色背景 (#e3f2fd)
booked: 橙色背景 (#fff3eb)
done: 绿色背景 (#e8f5e9)
```

**交互增强**：
- 订单卡片点击跳转到详情页
- 增加 `goDetail()` 方法

### 2. 课程订单详情页面 (`order-detail-course.vue`)

**页面结构**：

1. **状态卡片**
   - 动态图标和文案
   - 根据订单状态显示不同背景色
   - 状态提示文案

2. **课程信息卡片**
   - 课程名称和类型（私教/团课）
   - 私教课：课包进度条、已用/总课时、有效期、课包状态
   - 团课：上课时间、地点、教练信息

3. **消课记录卡片**（仅私教课）
   - 按时间倒序展示消课记录
   - 显示消课时间、教练、出勤状态
   - 序号标识

4. **订单信息卡片**
   - 订单编号（可复制）
   - 下单时间、支付时间、完成时间等
   - 订单状态、支付方式、备注

5. **费用明细卡片**
   - 课程原价
   - 优惠券抵扣
   - 实付金额

6. **退款信息卡片**（退款相关状态显示）
   - 退款原因
   - 退款金额

**底部操作栏**：
- 待支付：取消订单、立即支付
- 进行中：申请退款、查看课包（私教）
- 已完成：查看课包（私教）
- 退款中：退款审核中（禁用状态）

**数据加载逻辑**：
```javascript
1. 加载订单基本信息 (getOrderDetail)
2. 加载课程信息 (getCourseDetail)
3. 根据课程类型加载：
   - 私教课：
     * 加载课包信息 (GET /checkin/package/list)
     * 加载消课记录 (GET /checkin/list)
   - 团课：
     * 加载排课信息 (getScheduleDetail)
```

### 3. 路由配置 (`pages.json`)

新增路由：
```json
{
  "path": "pages/order/order-detail-course",
  "style": {
    "navigationStyle": "custom"
  }
}
```

## 状态流转说明

### 课程订单状态

| 状态值 | 状态名称 | 说明 | 用户操作 |
|--------|---------|------|---------|
| 1 | 待支付 | 订单已创建，等待支付 | 取消订单、立即支付 |
| 2 | 已支付 | 订单已支付，等待排课 | 申请退款 |
| 3 | 进行中 | 私教课包使用中/团课已预约 | 申请退款、查看课包 |
| 4 | 已完成 | 私教课包全部消课完成/团课已结束 | 查看课包 |
| 5 | 已取消 | 订单已取消 | - |
| 6 | 退款中 | 退款申请审核中 | - |
| 7 | 已退款 | 退款已完成 | - |
| 8 | 退款驳回 | 退款申请被驳回 | - |

### 课包状态

| 状态值 | 状态名称 | 颜色 |
|--------|---------|------|
| 0 | 已过期 | 灰色 |
| 1 | 进行中 | 橙色 |
| 2 | 已退费 | 灰色 |
| 3 | 已完成 | 绿色 |

## 视觉设计

### 状态卡片背景色

- 默认（待支付/已支付/进行中）：橙色渐变 `#9c3f00 → #ff7a2f`
- 已完成：绿色渐变 `#2e7d32 → #66bb6a`
- 已取消/已退款：灰色渐变 `#616161 → #9e9e9e`
- 退款中：橙色渐变 `#e65100 → #ff9800`
- 退款驳回：红色渐变 `#b71c1c → #ef5350`

### 课包进度条

- 背景：`rgba(156,63,0,0.1)`
- 填充：橙色渐变 `#9c3f00 → #ff7a2f`
- 高度：12rpx
- 圆角：999rpx

### 消课记录

- 序号圆圈：橙色背景 `#fff3eb`，橙色文字 `#9c3f00`
- 已出勤：绿色背景 `#e8f5e9`，绿色文字 `#4caf50`
- 缺勤：灰色背景 `#f5f6f7`，灰色文字 `#999`

## API 依赖

### 前端 API

- `getOrderDetail(id)` - 订单详情
- `getCourseDetail(id)` - 课程详情
- `getScheduleDetail(id)` - 排课详情
- `cancelOrder(id)` - 取消订单
- `payOrder(id)` - 支付订单
- `GET /checkin/package/list` - 课包列表
- `GET /checkin/list` - 消课记录列表

### 后端 API

- `GET /order/detail/{id}` - 订单详情
- `GET /course/detail/{id}` - 课程详情
- `GET /course/schedule/detail/{id}` - 排课详情
- `GET /checkin/package/list` - 课包列表
- `GET /checkin/list` - 消课记录列表

## 测试场景

### 场景 1：私教课订单详情

1. 创建并支付一个私教课订单
2. 进入订单详情页，验证：
   - 课程信息正确显示
   - 课包进度显示 0/12
   - 课包状态为"进行中"
   - 底部显示"申请退款"和"查看课包"按钮
3. 消课 6 节后，验证：
   - 课包进度更新为 6/12
   - 消课记录列表显示 6 条记录
   - 订单状态仍为"进行中"
4. 消课完 12 节后，验证：
   - 课包进度显示 12/12
   - 课包状态变为"已完成"
   - 订单状态变为"已完成"
   - 状态卡片背景变为绿色

### 场景 2：团课订单详情

1. 创建并支付一个团课订单
2. 进入订单详情页，验证：
   - 课程信息正确显示
   - 排课信息显示上课时间、地点、教练
   - 不显示课包进度和消课记录
   - 底部显示"申请退款"按钮
3. 团课结课后，验证：
   - 订单状态变为"已完成"
   - 状态卡片背景变为绿色

### 场景 3：订单列表状态筛选

1. 创建多个不同状态的订单
2. 在订单列表页切换状态标签，验证：
   - "全部"显示所有订单
   - "待支付"只显示状态 1 的订单
   - "进行中"只显示状态 3 的订单
   - "已完成"只显示状态 4 的订单
   - "已取消"只显示状态 5 的订单

### 场景 4：订单操作

1. 待支付订单：点击"取消订单"，验证订单状态变为已取消
2. 待支付订单：点击"立即支付"，验证支付成功后状态变为进行中
3. 进行中订单：点击"申请退款"，跳转到退款页面
4. 进行中订单（私教）：点击"查看课包"，跳转到课包列表页

## 注意事项

1. **课包查询**：通过 `orderId` 匹配课包时，需要注意类型转换（String 比较）
2. **消课记录排序**：后端默认按 `checkinTime` 倒序返回
3. **团课排课信息**：需要确保 `scheduleId` 存在才加载排课详情
4. **状态文案**：根据课程类型（私教/团课）显示不同的状态提示
5. **权限控制**：订单详情需要验证用户登录状态

## 后续优化建议

4. **评价功能**：已完成订单支持评价课程和教练
